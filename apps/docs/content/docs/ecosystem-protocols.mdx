## MCP (Model Context Protocol)

MCP standardizes how an agent connects to tools and context providers. OpenOS exposes MCP tools as normal `ToolDefinition` values so they work with `defineAgent` and the existing executor.

```typescript
import { defineAgent, loadMcpTools } from '@open-os/sdk'

const mcp = await loadMcpTools({
  command: 'npx',
  args: ['-y', '@modelcontextprotocol/server-everything'],
})

const agent = defineAgent({
  id: 'mcp-demo',
  name: 'MCP Demo',
  description: 'Uses MCP-backed tools.',
  model: { provider: 'anthropic', model: 'claude-sonnet-4-6' },
  tools: mcp.tools,
  systemPrompt: 'Use MCP tools when helpful.',
})

// When finished:
await mcp.close()
```

MCP I/O lives in `@open-os/mcp`, not in the kernel, to keep the kernel free of ad-hoc network or stdio coupling.

## A2A (Agent2Agent)

The [Agent2Agent protocol](https://developers.googleblog.com/en/a2a-a-new-era-of-agent-interoperability/) complements MCP: agents coordinate across runtimes using agent cards and task APIs.

```typescript
import { a2aDelegateRun, createA2aDelegateTool, fetchAgentCard } from '@open-os/sdk'

const card = await fetchAgentCard('https://example.com/.well-known/agent-card.json')
const out = await a2aDelegateRun({
  card,
  rpcUrl: 'https://example.com/rpc',
  input: 'Hello from OpenOS',
})

const remoteTool = createA2aDelegateTool({
  name: 'ask_remote',
  description: 'Ask the remote A2A agent.',
  cardUrl: 'https://example.com/.well-known/agent-card.json',
  rpcUrl: 'https://example.com/rpc',
})
```

Implementations live in `@open-os/a2a` and are re-exported from `@open-os/sdk`. Pass `rpcUrl` when the Agent Card does not advertise a JSON-RPC URL your build can parse.
