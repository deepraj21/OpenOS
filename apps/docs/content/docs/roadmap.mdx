## Phase 1 — Foundation (complete)

Monorepo, kernel, SDK, CLI (`run`, `list`, `init`), adapters, reference agents, Vitest suite.

## Phase 2 — Ecosystem (in progress)

- Docs site (Fumadocs + MDX).
- Registry + `publish` / `install`.
- Composer export pipeline.
- MCP bridge (`@openos/mcp`).
- A2A client.
- Python thin client for registry HTTP API.
- Open beta checklist (public registry instance, rate limits, comms).

## Phase 3 — Scale

Hosted runtime, observability, workspaces, analytics, enterprise features.
